DOM and interactions with the DOM.

The DOM is a like window into a website, as you look into this window you'll see the inner workings and structures of how the whole website works. You can not only look into to this window, but you can reach in and make adjustments inside this window which has an effect on the website. Now that is a oversimplied exaplanation of the DOM to my 5 year old cousin, but this is actually what the DOM is for. Basically a interface for developers to peer into the inner workings of a website, and make adjustments as required or to run design for a element of the website

Accessing Data from Arrays and Objects.

To my 5 year old cousin I would describe Objects as “things” like person, a car, a building, a book, a character in a game — basically anything that is made up or can be defined by a set of characteristics, while Arrays create and store lists of data (like a shopping list). To access these we can use both dot and bracket notation in our computer which allows us to add, change, and remove items from an object, while for the arrays we can use numbers from 0-9 allows us access and alter items in an array. To summarize to my little cousin access objects we use dots and brackets and access arrays we use numers 0-9.
